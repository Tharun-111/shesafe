# 🚀 SheSafe Deployment Guide

## Prerequisites
- GitHub account with the repo pushed
- MongoDB Atlas account (free tier)
- Render account (free tier)
- Vercel account (free tier)

---

## **Phase 1: Database Setup - MongoDB Atlas** 🍃

### 1. Create MongoDB Atlas Account
- Go to: https://www.mongodb.com/cloud/atlas
- Sign up for free
- Create a new project called "SheSafe"

### 2. Create a Cluster
- Select **M0 (Free Forever)** cluster tier
- Choose your preferred region (closest to users)
- Wait for cluster to initialize (~3-5 min)

### 3. Create Database User
- Go to **Database Access**
- Click **Add New Database User**
- Username: `shesafe_user`
- Password: (auto-generate a strong password and save it)
- Select **Autogenerated secure password**
- User Privileges: **Read and write to any database**
- Click **Add User**

### 4. Whitelist IP Address
- Go to **Network Access** → **Add IP Address**
- Select **Allow Access from Anywhere** (0.0.0.0/0)
- Click **Confirm**

⚠️ **Note**: For production, whitelist only Render's IP range

### 5. Get Connection String
- Go to **Clusters** → Click **Connect**
- Choose **Drivers** → **Node.js**
- Copy the connection string
- Format: `mongodb+srv://shesafe_user:PASSWORD@cluster0.xxxxx.mongodb.net/shesafe?retryWrites=true&w=majority`
- Replace `<password>` with your actual password
- Save this - you'll need it for Render

## **Phase 2: Deploy Backend to Render** 🚀

### 1. Connect Render to GitHub
- Go to: https://render.com
- Sign up and click **New** → **Web Service**
- Click **Connect GitHub Account**
- Search for `shesafe` repo
- Select it

### 2. Configure Web Service
- **Name**: `shesafe-backend`
- **Environment**: `Node`
- **Region**: `Oregon` (or closest to you)
- **Branch**: `main`
- **Build Command**: `cd server && npm install`
- **Start Command**: `cd server && node server.js`
- **Plan**: `Free`

### 3. Add Environment Variables
Click **Advanced** and add:

```
PORT=5000
NODE_ENV=production
JWT_SECRET=your_super_secret_jwt_key_change_me_in_production
MONGO_URI=mongodb+srv://shesafe_user:PASSWORD@cluster0.xxxxx.mongodb.net/shesafe?retryWrites=true&w=majority
CLIENT_URL=https://shesafe.vercel.app
```

Replace:
- `PASSWORD` with your MongoDB password
- `cluster0.xxxxx` with your actual cluster info
- `shesafe.vercel.app` with your actual Vercel URL (get this after deploying frontend)

### 4. Deploy
- Click **Create Web Service**
- Wait for deployment to complete (5-10 min)
- Copy your backend URL: `https://your-shesafe-backend.onrender.com`

---

## **Phase 3: Deploy Frontend to Vercel** ✨

### 1. Connect Vercel to GitHub
- Go to: https://vercel.com
- Sign up and click **Import Project**
- Select **GitHub** and search for `shesafe`
- Select the repo

### 2. Configure Project
- **Framework Preset**: `Create React App`
- **Root Directory**: `./client`
- **Build Command**: `npm run build` ✅ (auto-detected)
- **Output Directory**: `build` ✅ (auto-detected)

### 3. Add Environment Variables
- Click **Environment Variables**
- Add:
  ```
  REACT_APP_API_URL=https://your-shesafe-backend.onrender.com
  ```

### 4. Deploy
- Click **Deploy**
- Wait for deployment (2-3 min)
- Once done, your URL will be: `https://your-project-name.vercel.app`

---

## **Phase 4: Update Backend CORS** 🔒

After getting your Vercel URL, update Render environment variables:

1. Go to Render dashboard → Select `shesafe-backend`
2. Click **Environment**
3. Update `CLIENT_URL`:
   ```
   CLIENT_URL=https://your-vercel-url.vercel.app
   ```
4. Service will auto-redeploy

## **Troubleshooting**

### "CORS Error" in Browser
- Check if `CLIENT_URL` in Render matches your Vercel URL
- Verify CORS is enabled in `server.js`

### "Cannot connect to MongoDB"
- Verify IP whitelist includes 0.0.0.0/0 or Render's IPs
- Check username/password in connection string
- Verify cluster exists in MongoDB Atlas

### "Build fails on Render"
- Check logs: Click service → **Logs** tab
- Ensure `server/package.json` has `"start"` script
- Try: `npm install && npm start`

### "404 on API calls"
- Verify `REACT_APP_API_URL` env var on Vercel
- Check backend is running on Render (green light)
- Verify API routes exist in `server/routes/`
